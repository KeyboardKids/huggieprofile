UX RESEARCH & STRATEGY

1. 프로젝트 배경 및 데이터 수집

내용: 대전광역시청 정책 주제의 일환으로 중리종합사회복지관 어르신들을 대상으로 심층 인터뷰를 진행하고 데이터를 수집함.

핵심 인사이트: 어르신들에게는 '치매 인식 개선'과 '사회적 소속감'이 매우 중요한 가치임이 도출됨. 동시에 "기억력이 예전 같지 않다고 느낀다"는 실질적인 인지적 어려움과 시력 저하 등의 신체적 제약을 발견함.

2. 인지 저하 사용자를 위한 UI 설계 전략

'기억력이 예전 같지 않다'고 느끼는 어르신들이 서비스(허기)를 이용할 때 길을 잃지 않도록 프론트엔드 관점을 디자인한다.

  • 단일 페이지(SPA) 지향: 복잡한 메뉴 이동을 지양하고, 허기 소개를 직관적으로 인지할 수 있는 구조 채택.
  • 명확한 위치 인지: 현재 내가 서비스의 어느 단계에 와 있는지 시각적으로 명확히 표시(Stepper 활용).
  • 직관적인 가이드: 직관적인 아이콘과 친절한 안내 문구 배치.

3. 자바스크립트 아키텍처 설계

위의 UI/UX 기획을 유연하게 제어하기 위한 초기 변수 및 렌더링 함수 스케치 코드입니다.

// 어르신의 인지 상태 및 UI 테마 설정을 담은 사용자 상태 객체 (JSON 구조 연계)
const userConfig = {
  fontSize: "large",          // 시력 저하 어르신을 위한 큰 글씨 설정
  highContrast: true,         // 고대비 테마 활성화 여부
  currentStep: "intro-huggie" // 현재 어르신이 머물고 있는 단일 페이지 내의 위치
};

// 현재 위치(Step)를 시각적으로 명확하게 표시해 주는 함수
function updateStepper(step) {
  const steps = document.querySelectorAll('.step-item');
  steps.forEach(item => {
    if (item.dataset.step === step) {
      item.classList.add('active'); // 현재 단계 강조 (시각적 소속감 및 인지 유도)
    } else {
      item.classList.remove('active');
    }
  });
}

4. 코드 설계 포인트 (Code Review)

  1. 상태 관리의 중심 (`userConfig`): 어르신의 신체 상태 데이터를 객체화하여 향후 JSON 포맷으로 서버/로컬 스토리지와 연동할 수 있도록 확장성을 확보함.
  2. 웹 접근성 제어 (`updateStepper`): DOM 조작을 최소화하면서도 고령층의 '인지적 길찾기' 문제를 해결하기 위해 시각적 dynamic class 부여 방식을 채택함.
